UNIT 01

Introduction to Artificial Intelligence

What Is Intelligence?

Intelligence is "the ability to learn and understand, to solve problems and to make decisions." When we ask whether a machine can be intelligent, we are essentially asking whether it can exhibit this ability.

Definition — Artificial Intelligence

AI is the science of making machines do things that would require intelligence if done by humans. — Margaret Boden (1977)

The Turing Test

Alan Turing (1950) proposed the imitation game to sidestep philosophical debate. Instead of asking "Can machines think?", ask: "Can a machine pass a behavioural test for intelligence?"

Phase 1: Interrogator ↔ (Man | Woman) — guess who is who Phase 2: Interrogator ↔ (Computer | Human) — if indistinguishable, machine passes Key rules: • Communication only through text terminals (no voice, no face) • Computer may deliberately slow down or make errors • Interrogator may ask anything • If fooled as often as a human, the computer passes
Key Points
  • Modern computers still cannot pass the full Turing Test.
  • The test provides a basis for evaluating narrow AI systems.
  • Human brain ≈ 1018 bits storage, processes at 1015 bits/sec.

History of AI — Four Eras

Era 1: Dark Ages & Birth (1943–1956)

  • McCulloch & Pitts (1943): First neural network model — binary neuron states.
  • von Neumann: Encouraged neural computing; ENIAC/EDVAC.
  • Shannon (1950): Chess has 10120 possible moves → need heuristics.
  • Dartmouth Workshop (1956): AI officially named as a science by McCarthy.

Era 2: Great Expectations (1956–Late 1960s)

  • McCarthy invented LISP; proposed the Advice Taker — the first knowledge-based system.
  • Rosenblatt proved the perceptron convergence theorem.
  • GPS (General Problem Solver) — first system separating problem-solving from data.
  • Zadeh (1965) published "Fuzzy Sets" — largely ignored by the West.

Era 3: Unfulfilled Promises (Late 1960s–Early 1970s)

Weak methods (general search, no domain knowledge) failed on real problems. Funding was cut ("AI Winter").

Era 4: Knowledge-Based Systems (1970s–Present)

  • DENDRAL (1971): Molecular structure analysis — first real expert system.
  • MYCIN: Medical diagnosis using backward chaining.
  • PROSPECTOR: Mineral exploration using Bayesian reasoning.
  • 1980s: Neural network revival. 1990s: Fuzzy logic boom in Japan (consumer products).
Know the three generations of AI researchers and the key systems from each era (DENDRAL, MYCIN, PROSPECTOR). These appear in essay questions.

UNIT 03

Rule-Based Expert Systems

Production Rules

Definition — Production Rule

A rule of the form: IF [antecedent/condition] THEN [consequent/action]. The most popular knowledge representation technique.

Example Rules
IF the season is autumn AND the sky is cloudy AND the wind is low
THEN forecast is drizzle {cf 1.0}; forecast is rain {cf 0.9}

IF patient has fever AND patient has rash
THEN infection is likely

Five Components of an Expert System

ComponentFunction
Knowledge BaseContains domain rules (IF-THEN)
Database (Working Memory)Set of facts about the current situation
Inference EngineLinks rules with facts; drives reasoning
Explanation FacilitiesExplains HOW a conclusion was reached and WHY data was needed
User InterfaceCommunication layer between user and system

Inference Techniques

Forward Chaining (Data-Driven)

Start from known facts → apply rules → derive new facts → repeat until no more rules fire. Used when we have data and want to see what can be concluded.

Known facts: A, B, C, D, E Rule 3: A → X ✓ fires → add X Rule 2: X ∧ B ∧ E → Y ✓ fires → add Y Rule 1: Y ∧ D → Z ✓ fires → add Z Result: Z is proven. Best for: analysis, interpretation (e.g., DENDRAL)

Backward Chaining (Goal-Driven)

Start from a goal (hypothesis) → find rules that conclude it → try to satisfy their conditions → recurse. Used when we have a specific goal to prove.

Goal: prove Z Pass 1: Find Rule 1: Y ∧ D → Z ← stack it; now need Y and D Pass 2: Find Rule 2: X ∧ B ∧ E → Y ← stack it; now need X, B, E Pass 3: Find Rule 3: A → X ← stack it; need A Pass 4: A found in database → fire Rule 3 → X proved Pass 5: B, E in database → fire Rule 2 → Y proved Pass 6: D in database → fire Rule 1 → Z proved ✓ Best for: diagnosis (e.g., MYCIN)
CriterionForward ChainingBackward Chaining
DirectionData → ConclusionGoal → Data
Starting pointKnown factsHypothesis/Goal
User inputNo new questionsMay ask user for missing data
Best forAnalysis, monitoringDiagnosis, classification
EfficiencyMay fire irrelevant rulesOnly fires relevant rules

Conflict Resolution

When multiple rules can fire in a cycle, the inference engine must choose one. This is called conflict resolution.

Three Main Strategies
  • Priority-based: Fire rule with highest declared priority.
  • Most specific rule: Fire the rule with the most conditions (longest match). More specific = more information processed.
  • Most recent data: Fire the rule whose conditions use the most recently added facts (using time tags).

Metaknowledge

Definition

Metaknowledge = knowledge about knowledge. Metarules control how task-specific rules are used. Example: "Rules from certified experts have higher priority than rules from trainees."

Advantages & Disadvantages

Advantages ✓Disadvantages ✗
Natural knowledge representationOpaque relations between large sets of rules
Uniform IF-THEN structureExhaustive search is inefficient at scale
Separation of knowledge from processingCannot learn from experience
Handles incomplete and uncertain dataKnowledge acquisition is slow and expensive
The MEDIA ADVISOR example from the book is a great way to understand how a real rule-based system works. Forward chaining fires all rules; backward chaining only fires what's needed to prove the goal.
UNIT 04

Uncertainty Management

Sources of Uncertainty

4 Main Sources (Bonissone & Tong, 1985)
  • Weak implications: Vague associations between conditions and conclusions.
  • Imprecise language: Words like "often," "rarely" have no exact numeric meaning.
  • Unknown data: Missing or incomplete information.
  • Conflicting experts: Different experts give contradictory rules.

Bayesian Reasoning

Based on conditional probability. Given evidence E, what is the probability hypothesis H is true?

Bayes' Rule
P(H|E) = P(E|H) × P(H) / [ P(E|H) × P(H) + P(E|¬H) × P(¬H) ] Where: P(H) = prior probability of H (before seeing evidence) P(E|H) = probability of observing E if H is true P(H|E) = posterior probability of H after observing E

How Bayesian Expert Systems Work

  • Expert provides prior probabilities for all hypotheses.
  • Expert provides conditional probabilities P(E|H) for each evidence-hypothesis pair.
  • User inputs observed evidence.
  • System calculates posterior probabilities and ranks hypotheses.
  • As more evidence is observed, probabilities update (accumulated evidence).
Bias of Bayesian method: Assumes conditional independence of evidence — rarely true in real life. Requires large numbers of probabilities from experts. This is why the CF approach was developed.

Certainty Factors (CF) Theory

Developed for MYCIN as a simpler alternative to full Bayesian reasoning. CF represents a degree of belief in a hypothesis.

CF Formula
CF(H, E) = MB(H, E) − MD(H, E) MB = Measure of Belief (how much evidence supports H) MD = Measure of Disbelief (how much evidence contradicts H) CF range: −1 (definitely false) to +1 (definitely true) Example Rule: IF fever AND stiff_neck THEN meningitis {cf 0.8}
FeatureBayesian ReasoningCertainty Factors
FoundationClassical probability theoryAd-hoc but practical
RequirementsMany prior probabilities neededJust CF values per rule
Independence assumptionRequiredNot required
Expert burdenVery highLower
Best forWhen good probability data existsWhen expert intuition guides design
Be able to apply Bayesian update formula to a simple numerical example — it's a common exam question. Also understand the difference between prior and posterior probability.
UNIT 05

Frame-Based Systems

What Is a Frame?

Definition — Frame (Minsky, 1975)

A data structure with typical knowledge about a particular object or concept. Think of it as a "record" or "object" that groups related attributes together.

Frame: PERSON ├── Name: [value] ├── Age: [value] ├── Height: [numeric range: 0–250 cm] ├── Weight: [numeric] └── Job: [value or procedure] Frame: COMPUTER (class) ├── Processor: [default: Intel i5] ├── RAM: [numeric, GB] ├── Price: [range: $750–$1500] ├── WHEN_NEEDED: [calculate warranty expiry] └── WHEN_CHANGED: [update stock count]

Key Concepts

Class vs. Instance

  • Class-frame: Describes a group of similar objects (e.g., "Computer").
  • Instance-frame: A specific object (e.g., "My IBM ThinkPad X1").

Inheritance

Instances automatically inherit all attributes and default values from their class. This supports code reuse and reflects the real-world concept of "is-a" relationships.

Three Types of Relationships
  • Generalisation (is-a): Car is-a Vehicle. Subclass inherits from superclass.
  • Aggregation (part-of): Engine is-part-of Car.
  • Association: Mr. Black owns Car, House, Computer (semantic link).

Facets

Extended information about a slot, including: value facets (default/initial), prompt facets (ask user), and inference facets (when to stop inference).

Methods and Demons

TypeWhen it firesBest for
WHEN CHANGEDWhen an attribute's value changesTriggering actions, updating linked data
WHEN NEEDEDWhen an attribute's value is needed but not setLazy evaluation, default fallback
DemonSame as methods but limited to IF-THENSimple reactive rules

Advantages of Frame Systems

Why use frames?
  • Natural grouping of related knowledge (avoids scattered facts).
  • Inheritance reduces redundancy.
  • Object-oriented — mirrors real-world entities.
  • Rules and procedures can be attached directly to frames.
  • Faster search: only look in the relevant frame, not the whole knowledge base.
The Buy Smart example (property advisor) shows how a real frame-based system uses demons to filter instances and WHEN CHANGED methods to navigate. Know this example well.
UNIT 06

Fuzzy Logic and Fuzzy Expert Systems

Why Fuzzy Logic?

Classical (Boolean) logic is crisp: something is either true (1) or false (0). But real-world concepts like "tall," "young," "hot," and "fast" are inherently fuzzy — there are no sharp boundaries.

Definition — Fuzzy Set (Zadeh, 1965)

A set with fuzzy boundaries. Each element belongs to the set with a degree of membership μ ∈ [0, 1], not just 0 or 1.

Membership Functions

A membership function maps a value to its degree of belonging to a fuzzy set.

Universe: Man's Height (cm) Fuzzy Sets: SHORT: μ = 1.0 for x ≤ 160; decreases to 0 at x = 180 AVERAGE: μ peaks at x = 170; drops off on both sides TALL: μ = 0 for x ≤ 170; increases to 1.0 at x = 190+ A man of 175 cm might have: μ(SHORT) = 0.1 μ(AVERAGE) = 0.7 μ(TALL) = 0.4 → He is somewhat average AND somewhat tall!

Linguistic Variables and Hedges

  • Linguistic variable: A variable described in words (e.g., "Speed" with values "Slow", "Medium", "Fast").
  • Hedges: Modifiers that change the degree of membership.
HedgeEffectMath
very tallConcentration — narrows the setμ2(x)
more or less tallDilation — broadens the setμ0.5(x)
indeed tallIntensificationBoosts above-0.5, reduces below-0.5

Fuzzy Set Operations

OperationClassicalFuzzy
Complement (NOT)1 − AμNOT A(x) = 1 − μA(x)
Intersection (AND)A ∩ BμA∩B(x) = min(μA(x), μB(x))
Union (OR)A ∪ BμA∪B(x) = max(μA(x), μB(x))

Fuzzy Inference Process — 4 Steps

INPUT: crisp value(s) ↓ Step 1: FUZZIFICATION Convert crisp inputs to degrees of membership in fuzzy sets. e.g., Speed = 60 km/h → μ(Medium) = 0.6, μ(Fast) = 0.2 Step 2: RULE EVALUATION Apply AND (min) / OR (max) to multi-antecedent rules. Fire each relevant rule and get a firing strength. e.g., IF speed is Medium AND road is Wet THEN brake is Hard Step 3: AGGREGATION Combine all rule outputs (union of clipped/scaled fuzzy sets) into a single output fuzzy region. Step 4: DEFUZZIFICATION Convert aggregate fuzzy set → single crisp output number. Most common: Centroid of Gravity (COG) method. ↓ OUTPUT: crisp value

Mamdani vs. Sugeno Methods

FeatureMamdani (1975)Sugeno (1985)
Rule consequentFuzzy setMathematical function or constant (singleton)
DefuzzificationComplex (centroid of 2D shape)Simple weighted average
Computational loadHigherLower, faster
Best forCapturing expert knowledge, human-interpretableControl systems, optimization, adaptive systems
OutputFuzzy set clipped/scaledSingleton spike (k = constant)

Building a Fuzzy Expert System — 5 Steps

  1. Specify the problem and define linguistic variables.
  2. Determine fuzzy sets and membership functions.
  3. Elicit and construct fuzzy rules.
  4. Encode sets, rules and inference in the system.
  5. Evaluate and tune the system (most laborious step).
Important tuning tips: Maintain 25–50% overlap between neighbouring fuzzy sets. Wide sets → rough performance. Tune by adjusting set shapes, adding new rules, or adjusting rule weights.
Be able to perform fuzzy inference on a simple example: given inputs, fuzzify them, evaluate 2–3 rules, aggregate, and defuzzify using centroid. This is a classic exam problem.
UNIT 07

Frame-Based Expert Systems (Applied)

This unit extends Unit 05 with practical application aspects, covering the integration of frames with rules and the development lifecycle.

Frames + Rules Together

In frame-based systems, rules play an auxiliary role. Frames store the main knowledge; rules help evaluate frame attributes dynamically.

How rules differ in frame-based systems
  • Rules use pattern-matching clauses with variables to find matching instance-frames.
  • The inference engine searches for the goal attribute across all frames.
  • SEARCH ORDER facet determines whether rules or WHEN NEEDED methods fire first.

Development Steps for Frame-Based Systems

  1. Specify the problem and define scope.
  2. Determine classes and their attributes.
  3. Define instances (link to databases if needed).
  4. Design displays (user interface).
  5. Define WHEN CHANGED / WHEN NEEDED methods and demons.
  6. Define rules for complex evaluations.
  7. Evaluate and expand the system.
Remember: frames are better than rule-only systems when objects have shared structure and hierarchy. They avoid searching the entire knowledge base for every query.
UNIT 08

Artificial Neural Networks

How the Brain Works

The human brain has ~10 billion neurons and 60 trillion synaptic connections. Neural networks process information in parallel, storing knowledge globally across the entire network — not at specific locations.

Key Insight

Neural networks learn by strengthening useful connections and weakening useless ones — exactly like the brain's plasticity. This enables learning from experience.

The Perceptron (Single Neuron)

Inputs: x₁, x₂, ..., xₙ Weights: w₁, w₂, ..., wₙ Net input: X = Σ(xᵢ × wᵢ) − θ (θ = threshold) Activation: Y = { 1 if X ≥ 0 { 0 if X < 0 (step function) Learning Rule: wᵢ(new) = wᵢ(old) + α × (desired − actual) × xᵢ (α = learning rate, e.g., 0.1)

Multilayer Neural Networks (MLP)

Architecture

  • Input layer: Receives external data (no processing).
  • Hidden layer(s): Extract features, apply transformations.
  • Output layer: Produces final predictions/classifications.

Backpropagation (Learning Algorithm)

FORWARD PASS: Input → Hidden layer(s) → Output Calculate error = Desired output − Actual output BACKWARD PASS: Propagate error backwards through network Adjust weights using gradient descent: Δwᵢⱼ = α × δⱼ × yᵢ (α=learning rate, δⱼ=error signal, yᵢ=layer i output) Repeat until error is minimized.

Network Architectures

Network TypeStructureBest for
PerceptronSingle layer, step activationLinearly separable problems
MLP (Backprop)Multiple layers, sigmoid/ReLUClassification, regression
HopfieldRecurrent, fully connectedAssociative memory, pattern completion
BAMBidirectional recurrentBidirectional associative memory

Hopfield Network

  • Fully connected recurrent network — each neuron connects to all others.
  • Stores patterns as stable energy states (attractors).
  • Acts like associative memory: present a noisy/partial pattern → retrieves the complete stored pattern.
  • Energy always decreases during recall → guaranteed convergence.

Self-Organising Neural Networks (Kohonen)

  • Unsupervised learning — no labels needed.
  • Competitive learning: neurons compete, winner (best-matching unit) and its neighbours are updated.
  • Creates a topographic map: similar inputs activate nearby neurons.
  • Useful for clustering, visualisation and data compression.

NN vs Expert Systems

FeatureNeural NetworkExpert System
LearningYes — from dataNo — manual rule entry
ExplanationNo — black boxYes — rule trace
Noise toleranceHighLow (needs exact match)
Knowledge representationDistributed weightsExplicit IF-THEN rules
Knowledge acquisitionAutomatic from training dataExpensive expert interviews
Backpropagation algorithm steps are very commonly tested. Know: forward pass → error calculation → backward pass → weight update. Also know why we need hidden layers (XOR problem cannot be solved by single-layer perceptron).
UNIT 09

Evolutionary Computing

The Biological Analogy

Evolution works through: reproduction, mutation, recombination (crossover), and natural selection of the fittest. Genetic algorithms simulate this to search for optimal solutions.

Genetic Algorithms (GA)

Definition — Genetic Algorithm (Holland, 1970s)

A stochastic search algorithm based on biological evolution. Represents solutions as binary "chromosomes" and evolves populations toward optimal solutions through selection, crossover and mutation.

GA Steps (10-Step Process)

Step 1: Encode problem as fixed-length binary chromosome. Choose population size N, crossover prob pc, mutation prob pm. Step 2: Define fitness function f(chromosome). Step 3: Generate initial random population x₁, x₂, ..., xN Step 4: Calculate fitness: f(x₁), f(x₂), ..., f(xN) Step 5: Select parent chromosomes (probability ∝ fitness). Step 6: Apply crossover and mutation to create offspring. Step 7: Place offspring in new population. Step 8: Repeat Steps 5–7 until new population = N. Step 9: Replace old population with new population. Step 10: Repeat from Step 4 until termination criterion met.

Genetic Operators

OperatorDescriptionTypical Rate
SelectionRoulette wheel — fitter chromosomes more likely chosen—
CrossoverTwo parents "break" at random point; swap tails to create offspringpc ≈ 0.7
MutationRandomly flip a bit in a chromosome (prevents local optima)pm ≈ 0.001
Crossover Example: Parent 1: 1 1 0 0 | 1 0 1 0 → Child 1: 1 1 0 0 1 1 1 1 Parent 2: 0 1 0 1 | 1 1 1 1 → Child 2: 0 1 0 1 1 0 1 0 ↑ crossover point Mutation Example: Original: 1 1 [0] 0 1 0 1 0 Mutated: 1 1 [1] 0 1 0 1 0 (bit 3 flipped)

Schema Theorem (Why GAs Work)

A schema is a template like 1 * * 0 (wildcards = *). Short, high-fitness schemata grow exponentially across generations. This is the theoretical foundation of GAs.

Evolution Strategies

  • Developed by Rechenberg & Schwefel (1963) for engineering optimisation.
  • Uses only mutation (no crossover) with Gaussian (normally distributed) perturbations.
  • Works directly on real numbers — no binary encoding needed.
  • Better than GAs for continuous parameter optimisation.

Genetic Programming

  • Extension of GAs by John Koza (1990s).
  • Evolves computer programs (LISP S-expressions), not just bit strings.
  • Programs are represented as trees; crossover swaps subtrees; mutation changes nodes.
  • Self-modifying code: "evolve the solution, not just the parameters."
MethodRepresentationOperatorsBest for
Genetic AlgorithmFixed-length binary stringSelection, Crossover, MutationDiscrete optimization
Evolution StrategyReal-valued vectorMutation only (Gaussian)Continuous optimization
Genetic ProgrammingVariable-length program treeCrossover, Mutation, CloningProgram synthesis, symbolic regression
Be able to trace through 1–2 generations of a simple GA example manually — selection (roulette wheel), crossover, and mutation. Also know the difference between crossover and mutation purposes.
UNIT 10

Hybrid Intelligent Systems

Why Hybrid Systems?

No single AI technique is perfect. Each has strengths and weaknesses. Combining them creates systems that are more powerful than the sum of their parts.

Soft Computing (Zadeh, 1991)

A collection of methodologies (fuzzy logic, neural networks, evolutionary computation) that collectively provide a framework for reasoning in uncertain and imprecise environments — mirroring human cognition.

TechnologyStrengthWeakness
Expert SystemsKnowledge representation, explanationNo learning, rigid, expensive to build
Fuzzy SystemsHandles imprecision, linguistic reasoningCannot learn or adapt automatically
Neural NetworksLearning, generalisation, noise toleranceBlack box, needs large training data
Genetic AlgorithmsGlobal optimization, adaptabilityComputationally expensive, slow

Neural Expert Systems

A hybrid combining neural networks (as knowledge base) with rule-based inference (as explanation layer).

Key Features
  • Neural network is trained on data → weights encode knowledge automatically.
  • Rule extraction unit converts weights to IF-THEN rules (explainability).
  • Approximate reasoning: can handle noisy/incomplete inputs (no exact match needed).
  • Rule extraction uses the heuristic: "Infer if known weighted input > sum of |unknown weights|."

Neuro-Fuzzy Systems

A neural network that is functionally equivalent to a fuzzy inference model. The neural structure provides learning; the fuzzy rules provide transparency.

Neuro-Fuzzy Architecture (5 Layers): Layer 1: Input layer — passes crisp values Layer 2: Fuzzification — membership functions as activation functions (parameters a, b act like weights → can be trained!) Layer 3: Fuzzy rules — product operation for AND (weights = rule certainty factors) Layer 4: Output membership — probabilistic OR for union Layer 5: Defuzzification — crisp output via weighted average

ANFIS — Adaptive Neuro-Fuzzy Inference System

  • Combines neuro-fuzzy with the Sugeno fuzzy model.
  • Uses hybrid learning: backpropagation for premise parameters + least squares for consequent parameters.
  • Computationally efficient and highly accurate.
  • Used widely in control, forecasting, and signal processing.

Evolutionary Neural Networks

Genetic algorithms are used to optimise the architecture and weights of neural networks — removing the need for manual design and backprop.

Fuzzy Evolutionary Systems

Evolutionary algorithms optimise fuzzy system parameters (membership function shapes, rule weights).

The key question is: "Which components combine to make which hybrid, and what does each contribute?" Neuro-fuzzy = NN + Fuzzy. ANFIS = NN + Sugeno Fuzzy. Neural Expert = NN + Rule-Based. Evolutionary NN = GA + NN.
UNIT 11

Knowledge Engineering

What Is Knowledge Engineering?

Definition

Knowledge engineering is the process of building knowledge-based systems. The knowledge engineer is responsible for acquiring knowledge from domain experts, representing it formally, and building and testing the system.

Six Phases of the Knowledge Engineering Process

#PhaseActivity
1Problem AssessmentIs this an appropriate problem for an intelligent system?
2Data and Knowledge AcquisitionInterview experts, review documents, observe task performance.
3Knowledge Base DevelopmentEncode rules, frames, or network weights.
4System PrototypingBuild and test a working prototype.
5System EvaluationValidate with real test cases; compare with human expert.
6System Integration and MaintenanceDeploy; update as knowledge evolves.

Choosing the Right Tool

Problem typeBest toolWhy
Rules and logic available from expertsRule-based Expert SystemDirect encoding of expertise
Vague, imprecise, linguistic knowledgeFuzzy Expert SystemHandles imprecision natively
Large numerical dataset, pattern recognitionNeural NetworkLearns from data, generalises
Optimisation, scheduling, searchGenetic AlgorithmHeuristic global search
Need both learning AND explanationHybrid (Neuro-Fuzzy, Neural Expert)Best of both worlds

Data Mining and Knowledge Discovery

Extracting useful patterns from large datasets. Techniques include decision trees (ID3, C4.5), neural network classifiers, and association rule mining. Decision trees are particularly human-readable and create IF-THEN rules automatically.

Key Knowledge Engineering Lessons
  • The most critical phase is problem assessment — not every problem needs an intelligent system.
  • Knowledge acquisition is the bottleneck — experts are hard to interview and may contradict each other.
  • An expert system is validated by comparing it to human experts in test consultations.
  • Hybrid systems are now the norm for real applications — no single technique solves everything.
